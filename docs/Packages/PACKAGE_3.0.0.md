# Package 3.0.0 - Read-Only ACE Data Foundation

## Purpose

Begin Phase 3 by giving ACEMD safe, read-only access patterns for ACE data before account, character, world, or database management actions are introduced.

## Scope

- ACE Data Service foundation.
- Read-only connection discovery for ACE auth, shard, and world databases.
- Administration module routes for Servers, Accounts, Characters, World, and Database.
- Schema inventory and capability reporting.
- Account and character discovery with search and row limits.
- World table discovery and initial counts.
- SQL read-only enforcement.

## Out of scope

- Account edits.
- Character edits.
- World edits.
- Password changes.
- Account creation.
- Public portal functionality.
- ACEMD authentication/RBAC.

## SQL migration

No SQL migration is required.
