** Boolean Settings (true/false) **

/modifybool <setting> <true/false>

        account_login_boots_in_use: if FALSE, oldest connection to account is not booted when new connection occurs (default is True)
        advanced_combat_pets: (non-retail function) If enabled, Combat Pets can cast spells (default is False)
        advocate_fane_auto_bestow: If enabled, Advocate Fane will automatically bestow new advocates to advocate_fane_auto_bestow_level (default is False)
        aetheria_heal_color: If enabled, changes the aetheria healing over time messages from the default retail red color to green (default is False)
        allow_door_hold: enables retail behavior where standing on a door while it is closing keeps the door as ethereal until it is free from collisions, effectively holding the door open for other players (default is True)
        allow_fast_chug: enables retail behavior where a player can consume food and drink faster than normal by breaking animation (default is True)
        allow_jump_loot: enables retail behavior where a player can quickly loot items while jumping, bypassing the 'crouch down' animation (default is True)
        allow_negative_dispel_resist: enables retail behavior where #-# negative dispels can be resisted (default is True)
        allow_negative_rating_curve: enables retail behavior where negative DRR from void dots didn't switch to the reverse rating formula, resulting in a possibly unintended curve that quickly ramps up as -rating goes down, eventually approaching infinity / divide by 0 for -100 rating. less than -100 rating would produce negative numbers. (default is True)
        allow_pkl_bump: enables retail behavior where /pkl checks for entry collisions, bumping the player position over if standing on another PKLite. This effectively enables /pkl door skipping from retail (default is True)
        allow_summoning_killtask_multicredit: enables retail behavior where a summoner can get multiple killtask credits from a monster (default is True)
        assess_creature_mod: (non-retail function) If enabled, re-enables former skill formula, when assess creature skill is not trained or spec'ed (default is False)
        chat_disable_general: disable general global chat channel (default is False)
        chat_disable_lfg: disable lfg global chat channel (default is False)
        chat_disable_olthoi: disable olthoi global chat channel (default is False)
        chat_disable_roleplay: disable roleplay global chat channel (default is False)
        chat_disable_trade: disable trade global chat channel (default is False)
        chat_echo_only: global chat returns to sender only (default is False)
        chat_echo_reject: global chat returns to sender on reject (default is False)
        chat_inform_reject: global chat informs sender on reason for reject (default is True)
        chat_log_abuse: log abuse chat (default is False)
        chat_log_admin: log admin chat (default is False)
        chat_log_advocate: log advocate chat (default is False)
        chat_log_allegiance: log allegiance chat (default is False)
        chat_log_audit: log audit chat (default is True)
        chat_log_debug: log debug chat (default is False)
        chat_log_fellow: log fellow chat (default is False)
        chat_log_general: log general chat (default is False)
        chat_log_global: log global broadcasts (default is False)
        chat_log_help: log help chat (default is False)
        chat_log_lfg: log LFG chat (default is False)
        chat_log_olthoi: log olthoi chat (default is False)
        chat_log_qa: log QA chat (default is False)
        chat_log_roleplay: log roleplay chat (default is False)
        chat_log_sentinel: log sentinel chat (default is False)
        chat_log_society: log society chat (default is False)
        chat_log_trade: log trade chat (default is False)
        chat_log_townchans: log advocate town chat (default is False)
        chat_requires_account_15days: global chat privileges requires accounts to be 15 days or older (default is False)
        chess_enabled: if FALSE then chess will be disabled (default is True)
        client_movement_formula: If enabled, server uses DoMotion/StopMotion self-client movement methods instead of apply_raw_movement (default is False)
        container_opener_name: If enabled, when a player tries to open a container that is already in use by someone else, replaces 'someone else' in the message with the actual name of the player (default is False)
        corpse_decay_tick_logging: If ENABLED then player corpse ticks will be logged (default is False)
        corpse_destroy_pyreals: If FALSE then pyreals will not be completely destroyed on player death (default is True)
        craft_exact_msg: If TRUE, and player has crafting chance of success dialog enabled, shows them an additional message in their chat window with exact % (default is False)
        creature_name_check: if enabled, creature names in world database restricts player names during character creation (default is True)
        creatures_drop_createlist_wield: If FALSE then Wielded items in CreateList will not drop. Retail defaulted to TRUE but there are currently data errors (default is False)
        equipmentsetid_enabled: enable this to allow adding EquipmentSetIDs to loot armor (default is True)
        equipmentsetid_name_decoration: enable this to add the EquipmentSet name to loot armor name (default is False)
        fastbuff: If TRUE, enables the fast buffing trick from retail. (default is True)
        fellow_busy_no_recruit: if FALSE, fellows can be recruited while they are busy, different from retail (default is True)
        fellow_kt_killer: if FALSE, fellowship kill tasks will share with the fellowship, even if the killer doesn't have the quest (default is True)
        fellow_kt_landblock: if TRUE, fellowship kill tasks will share with landblock range (192 distance radius, or entire dungeon) (default is False)
        fellow_quest_bonus: if TRUE, applies EvenShare formula to fellowship quest reward XP (300% max bonus, defaults to false in retail) (default is False)
        gateway_ties_summonable: if disabled, players cannot summon ties from gateways. defaults to enabled, as in retail (default is True)
        house_15day_account: if disabled, houses can be purchased with accounts created less than 15 days old (default is True)
        house_30day_cooldown: if disabled, houses can be purchased without waiting 30 days between each purchase (default is True)
        house_hook_limit: if disabled, house hook limits are ignored (default is True)
        house_hookgroup_limit: if disabled, house hook group limits are ignored (default is True)
        house_per_char: if TRUE, allows 1 house per char instead of 1 house per account (default is False)
        house_purchase_requirements: if disabled, requirements to purchase/rent house are not checked (default is True)
        house_rent_enabled: If FALSE then rent is not required (default is True)
        iou_trades: (non-retail function) If enabled, IOUs can be traded for objects that are missing in DB but added/restored later on (default is False)
        item_dispel: if enabled, allows players to dispel items. defaults to end of retail, where item dispels could only target creatures (default is False)
        legacy_loot_system: use the previous iteration of the ace lootgen system (default is False)
        lifestone_broadcast_death: if true, player deaths are additionally broadcast to other players standing near the destination lifestone (default is True)
        loot_quality_mod: if FALSE then the loot quality modifier of a Death Treasure profile does not affect loot generation (default is True)
        npc_hairstyle_fullrange: if TRUE, allows generated creatures to use full range of hairstyles. Retail only allowed first nine (0-8) out of 51 (default is False)
        olthoi_play_enabled: allows players to create and play as olthoi characters (default is False)
        override_encounter_spawn_rates: if enabled, landblock encounter spawns are overidden by double properties below. (default is False)
        permit_corpse_all: If TRUE, /permit grants permittees access to all corpses of the permitter. Defaults to FALSE as per retail, where /permit only grants access to 1 locked corpse (default is False)
        persist_movement: If TRUE, persists autonomous movements such as turns and sidesteps through non-autonomous server actions. Retail didn't appear to do this, but some players may prefer this. (default is False)
        pet_stow_replace: pet stowing for different pet devices becomes a stow and replace. defaults to retail value of false (default is False)
        player_config_command: If enabled, players can use /config to change their settings via text commands (default is False)
        player_receive_immediate_save: if enabled, when the player receives items from an NPC, they will be saved immediately (default is False)
        pk_server: set this to TRUE for darktide servers (default is False)
        pk_server_safe_training_academy: set this to TRUE to disable pk fighting in training academy and time to exit starter town safely (default is False)
        pkl_server: set this to TRUE for pink servers (default is False)
        quest_info_enabled: toggles the /myquests player command (default is False)
        rares_real_time: allow for second chance roll based on an rng seeded timestamp for a rare on rare eligible kills that do not generate a rare, rares_max_seconds_between defines maximum seconds before second chance kicks in (default is True)
        rares_real_time_v2: chances for a rare to be generated on rare eligible kills are modified by the last time one was found per each player, rares_max_days_between defines maximum days before guaranteed rare generation (default is False)
        runrate_add_hooks: if TRUE, adds some runrate hooks that were missing from retail (exhaustion done, raise skill/attribute (default is False)
        reportbug_enabled: toggles the /reportbug player command (default is False)
        require_spell_comps: if FALSE, spell components are no longer required to be in inventory to cast spells. defaults to enabled, as in retail (default is True)
        safe_spell_comps: if TRUE, disables spell component burning for everyone (default is False)
        salvage_handle_overages: in retail, if 2 salvage bags were combined beyond 100 structure, the overages would be lost (default is False)
        show_ammo_buff: shows active enchantments such as blood drinker on equipped missile ammo during appraisal (default is False)
        show_aura_buff: shows active aura enchantments on wielded items during appraisal (default is False)
        show_dat_warning: if TRUE, will alert player (dat_warning_msg) when client attempts to download from server and boot them from game, disabled by default (default is False)
        show_dot_messages: enabled, shows combat messages for DoT damage ticks. defaults to disabled, as in retail (default is False)
        show_first_login_gift: if TRUE, will show on first login that the player earned bonus item (Blackmoor's Favor and/or Asheron's Benediction), disabled by default because msg is kind of odd on an emulator (default is False)
        show_mana_conv_bonus_0: if disabled, only shows mana conversion bonus if not zero, during appraisal of casting items (default is True)
        smite_uses_takedamage: if enabled, smite applies damage via TakeDamage (default is False)
        spellcast_recoil_queue: if true, players can queue the next spell to cast during recoil animation (default is False)
        spell_projectile_ethereal: broadcasts all spell projectiles as ethereal to clients only, and manually send stop velocity on collision. can fix various issues with client missing target id. (default is False)
        suicide_instant_death: if enabled, @die command kills player instantly. defaults to disabled, as in retail (default is False)
        taboo_table: if enabled, taboo table restricts player names during character creation (default is True)
        tailoring_intermediate_uieffects: If true, tailoring intermediate icons retain the magical/elemental highlight of the original item (default is False)
        trajectory_alt_solver: use the alternate trajectory solver for missiles and spell projectiles (default is False)
        universal_masteries: if TRUE, matches end of retail masteries - players wielding almost any weapon get +5 DR, except if the weapon "seems tough to master". if FALSE, players start with mastery of 1 melee and 1 ranged weapon type based on heritage, and can later re-select these 2 masteries (default is True)
        use_generator_rotation_offset: enables or disables using the generator's current rotation when offseting relative positions (default is True)
        use_turbine_chat: enables or disables global chat channels (General, LFG, Roleplay, Trade, Olthoi, Society, Allegience) (default is True)
        use_wield_requirements: disable this to bypass wield requirements. mostly for dev debugging (default is True)
        version_info_enabled: toggles the /aceversion player command (default is False)
        vendor_shop_uses_generator: enables or disables vendors using generator system in addition to createlist to create artificial scarcity (default is False)
        world_closed: enable this to startup world as a closed to players world (default is False)
		
** LONG Value Settings (Integers) **
/modifylong <setting> <value>

        char_delete_time: the amount of time in seconds a deleted character can be restored (default is 3600)
        chat_requires_account_time_seconds: the amount of time in seconds an account is required to have existed for for global chat privileges (default is 0)
        chat_requires_player_age: the amount of time in seconds a player is required to have played for global chat privileges (default is 0)
        chat_requires_player_level: the level a player is required to have for global chat privileges (default is 0)
        corpse_spam_limit: the number of corpses a player is allowed to leave on a landblock at one time (default is 15)
        default_subscription_level: retail defaults to 1, 1 = standard subscription (same as 2 and 3), 4 grants ToD pre-order bonus item Asheron's Benediction (default is 1)
        fellowship_even_share_level: level when fellowship XP sharing is no longer restricted (default is 50)
        mansion_min_rank: overrides the default allegiance rank required to own a mansion (default is 6)
        max_chars_per_account: retail defaults to 11, client supports up to 20 (default is 11)
        pk_timer: the number of seconds where a player cannot perform certain actions (ie. teleporting) after becoming involved in a PK battle (default is 20)
        player_save_interval: the number of seconds between automatic player saves (default is 300)
        rares_max_days_between: for rares_real_time_v2: the maximum number of days a player can go before a rare is generated on rare eligible creature kills (default is 45)
        rares_max_seconds_between: for rares_real_time: the maximum number of seconds a player can go before a second chance at a rare is allowed on rare eligible creature kills that did not generate a rare (default is 5256000)
        summoning_killtask_multicredit_cap: if allow_summoning_killtask_multicredit is enabled, the maximum # of killtask credits a player can receive from 1 kill (default is 2)
        teleport_visibility_fix: Fixes some possible issues with invisible players and mobs. 0 = default / disabled, 1 = players only, 2 = creatures, 3 = all world objects (default is 0)
		
** DOUBLE Value Settings (Decimal/Percent) **
/modifydouble <setting> <value>

        cantrip_drop_rate: Scales the chance for cantrips to drop in each tier. Defaults to 1.0, as per end of retail (default is 1)
        minor_cantrip_drop_rate: Scales the chance for minor cantrips to drop, relative to other cantrip levels in the tier. Defaults to 1.0, as per end of retail (default is 1)
        major_cantrip_drop_rate: Scales the chance for major cantrips to drop, relative to other cantrip levels in the tier. Defaults to 1.0, as per end of retail (default is 1)
        epic_cantrip_drop_rate: Scales the chance for epic cantrips to drop, relative to other cantrip levels in the tier. Defaults to 1.0, as per end of retail (default is 1)
        legendary_cantrip_drop_rate: Scales the chance for legendary cantrips to drop, relative to other cantrip levels in the tier. Defaults to 1.0, as per end of retail (default is 1)
        advocate_fane_auto_bestow_level: the level that advocates are automatically bestowed by Advocate Fane if advocate_fane_auto_bestow is true (default is 1)
        aetheria_drop_rate: Modifier for Aetheria drop rate, 1 being normal (default is 1)
        chess_ai_start_time: the number of seconds for the chess ai to start. defaults to -1 (disabled) (default is -1)
        encounter_delay: the number of seconds a generator profile for regions is delayed from returning to free slots (default is 1800)
        encounter_regen_interval: the number of seconds a generator for regions at which spawns its next set of objects (default is 600)
        fast_missile_modifier: The speed multiplier applied to fast missiles. Defaults to retail value of 1.2 (default is 1.2)
        ignore_magic_armor_pvp_scalar: Scales the effectiveness of IgnoreMagicArmor (ie. hollow weapons) in pvp battles. 1.0 = full effectiveness / ignore all enchantments on armor (default), 0.5 = half effectiveness / use half enchantments from armor, 0.0 = no effectiveness / use full enchantments from armor (default is 1)
        ignore_magic_resist_pvp_scalar: Scales the effectiveness of IgnoreMagicResist (ie. hollow weapons) in pvp battles. 1.0 = full effectiveness / ignore all resistances from life enchantments (default), 0.5 = half effectiveness / use half resistances from life enchantments, 0.0 = no effectiveness / use full resistances from life enchantments (default is 1)
        luminance_modifier: Scales the amount of luminance received by players (default is 1)
        melee_max_angle: for melee players, the maximum angle before a TurnTo is required. retail appeared to have required a TurnTo even for the smallest of angle offsets. (default is 0)
        mob_awareness_range: Scales the distance the monsters become alerted and aggro the players (default is 1)
        pk_new_character_grace_period: the number of seconds, in addition to pk_respite_timer, that a player killer is set to non-player killer status after first exiting training academy (default is 300)
        pk_respite_timer: the number of seconds that a player killer is set to non-player killer status after dying to another player killer (default is 300)
        quest_mindelta_rate: scales all quest min delta time between solves, 1 being normal (default is 1)
        rare_drop_rate_percent: Adjust the chance of a rare to spawn as a percentage. Default is 0.04, or 1 in 2,500. Max is 100, or every eligible drop. (default is 0.04)
        spellcast_max_angle: for advanced player spell casting, the maximum angle to target release a spell projectile. retail seemed to default to value of around 20, although some players seem to prefer a higher 45 degree angle (default is 20)
        trophy_drop_rate: Modifier for trophies dropped on creature death (default is 1)
        unlocker_window: The number of seconds a player unlocking a chest has exclusive access to first opening the chest. (default is 10)
        vendor_unique_rot_time: the number of seconds before unique items sold to vendors disappear (default is 300)
        vitae_penalty: the amount of vitae penalty a player gets per death (default is 0.05)
        vitae_penalty_max: the maximum vitae penalty a player can have (default is 0.4)
        void_pvp_modifier: Scales the amount of damage players take from Void Magic. Defaults to 0.5, as per retail. For earlier content where DRR isn't as readily available, this can be adjusted for balance. (default is 0.5)
        xp_modifier: scales the amount of xp received by players (default is 1)
