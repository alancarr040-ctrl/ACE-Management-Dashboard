# Documentation Standards

## General

- Be concise.
- Every sentence should add information.
- Avoid unnecessary whitespace.
- Avoid decorative formatting.
- Prefer tables over prose for structured information.
- Prefer bullet lists over paragraphs.
- One topic per document.

## Headings

Use standard Markdown headings.

Good:

## Docker

Bad:

==================================
Docker
==================================

## Code

Only include code that is necessary to explain a procedure.

## Tables

Use tables when comparing configuration or settings.

## Philosophy

Documentation exists to reduce recovery time and eliminate ambiguity.
